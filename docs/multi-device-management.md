# Управление проектом с нескольких устройств (Multi-Device Setup & Remote Control)

Этот документ является **source of truth** по настройке окружения на нескольких компьютерах (например, перенос работы с основного Mac на Windows-компьютер или ноутбук) и удаленному управлению сборкой видеоуроков через облачный контур GitHub Actions.

---

## 1. Синхронизация кода через Git

Все рабочие файлы проекта, документация, скрипты и конфигурации хранятся в репозитории на GitHub:
* **Repository remote URL:** `https://github.com/webpot-ru/luna.git`
* **Основная ветка:** `main`

### 💡 Правила работы:
1. **Всегда делайте `git pull`** перед началом работы на любом компьютере, чтобы подтянуть последние изменения, сделанные на других устройствах.
2. **Делайте `git push`** сразу после того, как завершили изменения в коде или документации, чтобы они были доступны на других ваших компьютерах.

---

## 2. Локальные секреты и переменные окружения

Секретные ключи и файлы конфигурации не коммитятся в репозиторий из соображений безопасности (они внесены в `.gitignore`). При развертывании проекта на новом компьютере их нужно перенести вручную или создать заново.

### 📋 Список необходимых секретов:

1. **Файл `.env.local`** (в корне проекта):
   Создайте его на основе `.env.example`. Главное поле — `DATABASE_URL` для подключения к Postgres.
   * *Для локальной БД:* `DATABASE_URL=postgresql://lunacards:lunacards@127.0.0.1:55433/lunacards`
   * *Для подключения к другой машине в локальной сети:* замените `127.0.0.1` на сетевой IP-адрес этой машины.

2. **Токен GitHub (`.secrets/github-token.txt`):**
   Необходим для удаленного запуска сборки видеоуроков через скрипты API GitHub.
   * **Как получить:**
     1. Зайдите на GitHub в настройки профиля: **Settings** -> **Developer Settings** -> **Personal Access Tokens** -> **Tokens (classic)**.
     2. Нажмите **Generate new token**. Укажите понятное имя (например, `LunaCards-Remote`).
     3. Выберите scope: обязательно отметьте пункт **`workflow`** (позволяет управлять Actions).
     4. Сгенерируйте токен, скопируйте его и сохраните в файл `.secrets/github-token.txt` на вашем компьютере.
     *(Скрипты автоматически прочитают токен из этого файла).*

3. **Секреты Google OAuth (для синхронизации гугл-таблиц):**
   * `.secrets/google-oauth-client.json` — credentials клиента OAuth.
   * `.secrets/google-oauth-token.json` — сохраненный авторизационный токен пользователя.
   *(Нужно перенести с основного компьютера, чтобы скрипты экспорта Sheets работали без повторной веб-авторизации).*

## 3. Синхронизация базы данных Postgres

Чтобы локальные базы данных на разных компьютерах содержали одинаковые данные по колодкам, картам и результатам QA, используйте встроенные скрипты для резервного копирования и восстановления.

### 📤 Создание резервной копии (Дамп)
На компьютере, где находятся актуальные данные, запустите в терминале:
```bash
scripts/db-dump.sh
```
Скрипт создаст файл дампа в папке `outputs/db/` с именем формата `lunacards_<timestamp>.sql`.

### 📥 Восстановление базы на новом компьютере
1. Скопируйте созданный `.sql` файл на новый компьютер.
2. Убедитесь, что запущен локальный контейнер Postgres (`docker-compose up -d`).
3. Запустите восстановление:
   ```bash
   # Для Windows PowerShell
   docker exec -i lunacards-postgres-1 psql -U lunacards -d lunacards < outputs/db/lunacards_XXXX.sql
   
   # Или через bash-скрипт (если доступно окружение bash)
   scripts/db-restore.sh outputs/db/lunacards_XXXX.sql
   ```

## 4. Удаленное управление сборкой видеоуроков через GitHub Actions

Сборка видео на 54 языка — ресурсоемкий процесс. Чтобы не нагружать локальный компьютер (особенно если это ноутбук), вы можете запускать компиляцию видеоуроков напрямую в облаке GitHub Actions и следить за процессом локально.

В репозитории настроены три облачных Workflow:
1. `build-test-single.yml` — Быстрый тест сборки одного видеоурока (целевой язык ES, поддержка RU).
2. `build-videos.yml` — Сборка видеоуроков для конкретных указанных языков поддержки (по умолчанию: UZ, AZ, KK, KA, HY).
3. `build-all-videos-matrix.yml` — Массовая сборка для всех оставшихся языков (50 языков параллельно в облачной матрице).

### 🚀 Запуск сборки в облаке (`trigger-workflow.mjs`)
Скрипт обращается к GitHub API и запускает нужный workflow.

**Формат команды:**
```bash
node scripts/trigger-workflow.mjs [set_id] [support_langs] [concurrency] [workflow_filename]
```

**Примеры запуска:**
```bash
# 1. Запустить одиночный тестовый видеоурок (для проверки рендерера и шрифтов)
node scripts/trigger-workflow.mjs home_kitchen_cookware_pilot_01 ES 2 build-test-single.yml

# 2. Собрать видеоуроки для 5 приоритетных языков (UZ, AZ, KK, KA, HY)
node scripts/trigger-workflow.mjs home_kitchen_cookware_pilot_01 "UZ,AZ,KK,KA,HY" 2 build-videos.yml

# 3. Запустить параллельную сборку матрицы для всех остальных 50 языков
node scripts/trigger-workflow.mjs home_kitchen_cookware_pilot_01 ALL 2 build-all-videos-matrix.yml
```

## 5. Мониторинг и логирование удаленных процессов

После того как вы отправили команду на запуск в облако, вы можете отслеживать ее прямо из локального терминала с помощью вспомогательных утилит.

### 📊 Проверка статуса выполнения (`check-runs.mjs`)
Скрипт проверяет состояние последней запущенной задачи в GitHub Actions, выводит список выполняемых шагов и последние 35 строк логов в реальном времени.

**Запуск:**
```bash
node scripts/check-runs.mjs
```

### 📋 Скачивание полных логов (`download-job-log.mjs`)
Скрипт скачивает полный текстовый лог последней задачи в локальный файл `outputs/job-log.txt` для удобного анализа ошибок или отладки.

**Запуск:**
```bash
# Скачать логи последней запущенной задачи
node scripts/download-job-log.mjs

# Скачать логи конкретной задачи по её ID (если необходимо)
node scripts/download-job-log.mjs 27521973958
```

## 6. Локальный запуск сборки видеоуроков (Альтернативный)

Если вам все же необходимо собрать видео локально на компьютере, убедитесь, что в системе установлен **FFmpeg** и путь к нему прописан в переменной окружения `PATH`.

* **Сборка одного видеоурока:**
  ```bash
  node scripts/build-deck-video.mjs --set home_kitchen_cookware_pilot_01 --target TH --support RU --transition flip --quiz-limit 3
  ```
* **Массовая локальная сборка всех языков колоды:**
  ```bash
  node scripts/build-all-deck-videos.mjs --set home_kitchen_cookware_pilot_01 --support RU --concurrency 2
  ```

Результаты сборки будут сохранены в папку `outputs/video-generator/`, а в файл реестра `docs/video-lessons-registry.md` автоматически добавится запись со статусом `Pending`.
