- On 2026-06-20 the project gained a dedicated Codex operations source-of-truth document at `docs/codex-operations.md`. It inspects Git-visible changed paths and safe project config/docs only; it does not read `.local`, `.env`, `.secrets`, OAuth tokens or client-secret contents. Required checks fail the command; optional video-localization timeout/failure is a warning unless `-- --strict` is passed. The workflow safety check has a wider timeout because dirty multimedia worktrees can make `git status --untracked-files=all` slow.
- Project-local Codex hooks are configured in `.codex/hooks.json`. On 2026-06-20 the hook trust review for `/Users/lali/Documents/LUNA2` was accepted through the local Codex CLI (`Trust all and continue`), and a repeat CLI start went directly to the normal LUNA2 screen without the hook review prompt. The hooks attempt to block obvious destructive shell commands and secret-printing commands, and run the workflow safety check at turn stop. Hooks are a guardrail, not a replacement for `AGENTS.md`, project docs, sandboxing, explicit user confirmation or external-service readback. The current desktop `exec_command` tool may still bypass project hooks; use the npm checks as explicit verification when in doubt.
